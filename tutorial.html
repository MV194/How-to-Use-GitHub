Here's a little bit about GitHub:

What is GitHub?

GitHub is a web-based interface that uses Git, the open source version control software that lets many people makes separate changes to web pages at the same time. GitHub lets you to create, store, change, merge, and collaborate on files or code. You can access the GitHub repository if you're a member of a team and see the most recent version in real-time. You can also make requests of another repository and discuss the amount of times to improve the project along the way. GitHub is even called "a social coding platform" because people can coordinate, share, and collaborate code across different environments. GitHub uses Git which is a tool that allows developers and others to use version control. In summary, GitHub is one of many web interfaces for using Git.

Who made GitHub?

The GitHub service was developed by Chris Wanstrath, P. J. Hyett, Tom Preston-Werner, and Scott Chacon using Ruby on Rails, and started in February 2008.

Why GitHub?

GitHub helps teams work together in the same documents and the same project at the same time. This will reduce the risk of double work and duplicates of the same work. Also, it helps increase the productivity time and helps get the job done faster. Developers can build code, track changes, and make solutions to problems that come along the way with GitHub. For non-developers they can use it to create, edit and update website content.
